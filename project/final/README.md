# Projet final

## Objectif

Assembler un démonstrateur d'audit lisible à partir des briques validées pendant la semaine: contrôles déterministes du Jour 4, sourcing du Jour 2 et synthèse avec validation humaine du Jour 5.

## Entrées requises

- un cas ou une exception cible
- un montant observé et ses paramètres de calcul
- une source documentaire de référence
- un utilisateur capable de valider ou rejeter la synthèse finale

## Sorties attendues

- un constat d'audit court
- une preuve ou une référence documentaire
- une limite ou hypothèse explicitée
- un statut final après revue humaine

## Critères d'évaluation

- le workflow reste compréhensible en lecture directe
- la logique d'exceptions du Jour 4 est bien réutilisée
- la source du Jour 2 est visible dans la synthèse
- la validation humaine du Jour 5 reste obligatoire
- aucune réindexation ni initialisation d'infrastructure n'apparaît dans le projet final

## Simplifications autorisées

- garder une seule exception représentative
- utiliser une seule source documentaire au lieu d'un corpus complet
- réduire le texte de synthèse à un court brouillon métier
- conserver une seule branche validée et une seule branche rejetée
- omettre toute brique MCP si elle n'apporte pas de valeur démonstrative sur le scénario choisi
